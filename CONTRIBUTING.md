# Contributing

Thanks for helping keep this list sharp and useful. A small set of rules keeps the signal high.

## What belongs here

- Production-grade tools with active maintenance (a commit within the last 12 months).
- Direct relevance to the autonomous-science stack: orchestration, agents, lab hardware, compute, tracking, verification, knowledge management, extraction, science-focused LLMs, RL, vector stores, pipelines, safety, or observability.
- Projects used in real self-driving lab or autonomous research deployments.

## What doesn't belong

- Abandoned projects (no commits in 18+ months), forks of existing entries, or unmaintained mirrors.
- Generic ML libraries with no clear science angle (e.g., a generic CRUD framework, a basic chart library).
- Low-effort self-promotion, paid placements, or duplicate entries across categories.

## How to add an entry

1. Fork this repository.
2. Edit the relevant table in `README.md`.
3. Open a pull request that includes:
   - Library name
   - One-line description (concrete, no marketing language)
   - Link (GitHub repo for OSS; official site for commercial/closed projects)
   - Category
   - One-sentence rationale for why it belongs

## PR checklist

- [ ] Entry is in the correct category table
- [ ] Description is one line, under ~100 characters, no marketing adjectives
- [ ] Link resolves and points to the canonical repo or site
- [ ] Project has had a commit in the last 12 months
- [ ] No duplicate of an existing entry

This project follows the [Contributor Covenant](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).
